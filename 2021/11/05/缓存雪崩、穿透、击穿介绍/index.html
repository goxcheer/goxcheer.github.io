缓存雪崩、穿透、击穿介绍

Posted on 2021-11-05 In NoSQL , Redis

缓存雪崩、穿透、击穿最简介绍……

缓存雪崩

什么是缓存雪崩

缓存雪崩是指由于某些原因（比如redis缓存服突然宕机或是redis缓存中key在某一时刻大面积失效），导致大量请求直接访问数据库导致数据库崩溃，从而导致整个应用系统崩溃，引发灾难。

预防和解决缓存雪崩

缓存雪崩一般分为事前，事中和事后几个阶段来预防和解决。

  • 事前：

    保证缓存缓存服务的高可用性，使用redis集群（redis cluster or redis sentinel）方式部署redis，并且部署在多个机房之间，避免单机房断电等因素导致问题。

  • 事中：

    针对redis缓存中的key在某一时刻大量失效，可以对不同的redis key设置不同的过期时间，来错开缓存过期，避免缓存在同一时间失效。
引入多级缓存，如使用ehcache本地缓存，在redis缓存宕机的情况下还能挡住一部分请求。
另外针对服务请求，需要做限流或熔断降级处理（比如sentinel或者hystrix），来设置每秒最多能有多少请求，从而避免数据库宕机。

  • 事后：
    针对redis宕机，根据redis持久化机制快速恢复缓存数据。
    针对缓存大量失效，迅速进行缓存预热。

缓存穿透

缓存穿透指的是同一时刻访问大量数据库不存在的key（如黑客攻击），这时候因为redis缓存也没有key，所以大量请求会打到数据库层，可能导致db宕机。

解决这种缓存穿透攻击也比较简单：

  • 每次请求从数据库中只要没查到，就写一个空值到缓存里去，比如 set key UNKNOWN。然后设置一个过期时间（过期时间需设置很短），这样的话，下次有相同的 key 来访问的时候，在缓存失效之前，都可以直接从缓存中取数据。

  • 布隆过滤器，布隆过滤器原理, 先将数据库中的数据预存到布隆过滤器BitMap，这样只有布隆过滤器中可能存在的数据才去访问db，否则可以不访问（须注意布隆过滤器没法删除数据）。

缓存击穿

缓存击穿，就是说某个 key 非常热点，访问非常频繁，处于集中式高并发访问的情况，当这个 key 在失效的瞬间，大量的请求就击穿了缓存，直接请求数据库，就像是在一道屏障上凿开了一个洞。

解决方式也很简单，
可以将热点数据设置为永远不过期；
或者基于 redis or zookeeper 实现互斥锁，等待第一个请求构建完缓存之后，再释放锁，进而其它请求才能通过该 key 访问数据。
