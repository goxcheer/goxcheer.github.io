java RMI概述

Posted on 2019-05-10 | In Java , RMI

这个东西现在来说已经比较老了，但是作为一个java早期的基础性知识点，是有必要学习研究一下的…..

RPC简介

RPC （Remote procedure call protocal） 可以用于一个进程调用另一个进程（很可能在另一个远程主机上）中的过程，从而提供了过程的分布能力。
RPC协议其实是一个规范。Dubbo、Thrif、RMI、Webservice、Hessain都是依据这种规范实现的。

RPC框架的实现，就是使网络协议和网络IO对于调用端和服务端来说是透明；

一个RPC框架包含的要素

RMI概述

RMI(remote method invocation) , 可以认为是RPC的java版本。
RMI（Remote Method Invocation）为远程方法调用，是允许运行在一个Java虚拟机的对象调用运行在另一个Java虚拟机上的对象的方法。
这两个虚拟机可以是运行在相同计算机上的不同进程中，也可以是运行在网络上的不同计算机中。

RMI使用的是JRMP（Java Remote Messageing Protocol）, JRMP是专门为java定制的通信协议。

RMI工作过程

桩/框架（Stub/Skeleton）层：客户端的桩和服务器端的框架；

远程引用（remote reference）层：处理远程引用行为

传送层（transport）：连接的建立和管理，以及远程对象的跟踪

调用步骤：

1，客户调用客户端辅助对象stub上的方法

2，客户端辅助对象stub打包调用信息（变量，方法名），通过网络发送给服务端辅助对象skeleton

3，服务端辅助对象skeleton将客户端辅助对象发送来的信息解包，找出真正被调用的方法以及该方法所在对象

4，调用真正服务对象上的真正方法，并将结果返回给服务端辅助对象skeleton

5，服务端辅助对象将结果打包，发送给客户端辅助对象stub

6，客户端辅助对象将返回值解包，返回给调用者

7，客户获得返回值
