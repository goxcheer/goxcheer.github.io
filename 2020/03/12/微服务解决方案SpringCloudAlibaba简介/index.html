微服务解决方案SpringCloudAlibaba简介

Posted on 2020-03-12 | In SpringCloud

之前学过cloud，现在系统的整理知识点。。。。。。

什么是SpringCloud

SpringCloud提供了一些可以让开发者快速构建微服务应用的工具，比如配置管理、服务注册与发现、熔断、路由等，这些服务可以在分布式环境下很好的工作。

SpringCloud致力于解决如下问题：

  • Distributed/versioned configuration 分布式及版本化配置
  • Service registration and discovery 服务注册与发现
  • Routing 服务调用
  • Loading balancing 负载均衡
  • Circuit Breakers 熔断器
  • Global locks 全局锁
  • Leadership election and cluster state Leader选举及集群状态
  • Distributed messaging 分布式消息

SpringCloud并不是一个全新的框架，只是把一些比较优秀的解决微服务架构常见问题的框架基于SpringCloud规范进行了整合，通过Springboot这个框架进行再次封装后屏蔽掉复杂的配置，给开发者提供良好的开箱即用的微服务开发体验。

SpringCloud版本简介

SpringCloud采用了伦敦地铁站的名字根据字母表结合时间顺序来定义各大版本，以往发布的版本排序如下：

  • Angel
  • Brixton
  • Camden
  • Edgware
  • Finchley
  • Greenwich
  • Hoxton(最新的版本)

SpringCloud所有的子项目都依赖SpringBoot框架，所以两者之间的版本存在着依赖和兼容关系。

SpringCloud生态下服务治理的解决方案主要有2个：SpringCloud Netflix和Spring Cloud Alibaba.

SpringCloud Netflix

SpringCloud Netflix主要为微服务架构下提供解决方案，包括以下组件：

  • Eureka 服务注册与发现
  • Zuul 服务网关
  • Ribbon 负载均衡
  • Feign 远程服务的客户端代理
  • Hystrix 断路器，提供服务熔断和限流功能
  • Hystrix Dashboard 监控面板
  • Turbine 将各个服务上的Hystrix监控信息进行统一聚合。

由于Netflix维护不利，所以这些组件短期仍然可以使用，但是长期显然不合适。

SpringCloud Alibaba

SprringCloud Alibaba是阿里巴巴集团下的开源组件，为微服务开发提供一站式的解决方案。

  • Sentinel 流量监控和服务降级
  • Nacos 服务注册与发现，分布式配置中心
  • RocketMQ 消息驱动
  • Seate 分布式事务
  • Dubbo RPC通信
  • OSS 阿里云对象存储

SpringCloud Alibaba的开源组件在还没有织入SpringCloud生态的时候已经在各大公司广泛使用了，并且也已经经历了大规模业务量的考验，所以是推荐使用的。
