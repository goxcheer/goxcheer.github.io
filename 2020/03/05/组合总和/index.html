组合总和

Posted on 2020-03-05 | In 算法 , 深度优先搜索 , 回溯法

组合总和，典型一类题目的通用解法……

组合总和I

题目描述

给定一个无重复元素的数组 candidates 和一个目标数 target ，找出 candidates 中所有可以使数字和为 target 的组合。

candidates 中的数字可以无限制重复被选取。

说明：

所有数字（包括 target）都是正整数。
解集不能包含重复的组合。
示例 1:

输入: candidates = [2,3,6,7], target = 7,
所求解集为:
[
[7],
[2,2,3]
]
示例 2:

输入: candidates = [2,3,5], target = 8,
所求解集为:
[
[2,2,2,2],
[2,3,3],
[3,5]
]

解题思路

这种解集长度不确定的一般都采用递归的方式，递归的过程就是构造一颗递归树，达到叶子节点返回，走过的路径就是采用深度优先搜索。

  • 蓝色结点表示：尝试找到组合之和为该数的所有组合，怎么找呢？逐个减掉候选数组中的元素即可；
  • 以 target = 7 为根结点，每一个分支做减法；
  • 减到 00 或者负数的时候，到了叶子结点；
  • 减到 00 的时候结算，这里 “结算” 的意思是添加到结果集；
  • 从根结点到叶子结点（必须为 0）的路径，就是题目要我们找的一个组合。

但是这样可能有一个问题就是，结果集会重复，我们需要保证前面的结果后面不会再出现，所以每次搜索的起始值必然>=上一次搜索的起始值。（剪枝）
所以我们可以优先给输入集进行排序操作。

JAVA实现

这里需要注意的是采用ArrayDeque在每次递归返回时移除此前添加的元素，这样保证在所有的深度搜索过程保证只用一个ArrayDeque，添加到结果中只需要做一次深度拷贝即可。

组合总和II

题目描述

给定一个数组 candidates 和一个目标数 target ，找出 candidates 中所有可以使数字和为 target 的组合。

candidates 中的每个数字在每个组合中只能使用一次。

说明：

所有数字（包括目标数）都是正整数。
解集不能包含重复的组合。
示例 1:

输入: candidates = [10,1,2,7,6,1,5], target = 8,
所求解集为:
[
[1, 7],
[1, 2, 5],
[2, 6],
[1, 1, 6]
]
示例 2:

输入: candidates = [2,5,2,1,2], target = 5,
所求解集为:
[
[1,2,2],
[5]
]

解题思路

组合总数II和I其实差不多，只是每个数字在每个组合中只能使用一次。这就要考虑例如数组为1，1，1，2，3，4这种情况，在上下层递归的过程值相同是可以使用的，只是在同一层递归的过程中，因为在Array[i] == Array[i+1]的前提下，其实i搜索的结果已经将i+1的包括了，所以需要剪枝。
